你的时间非常值钱-HMTL，完美适配某公司的弹性上下班机制，让你知道还要干多久，同时让你知道你的时间有多值钱。

让我们先明确一下打卡的规则:每天的5:00-次日的5:00记为一天，一天被分为多个时间段。
1. Work 这是一个连续的时间段，默认为 08:00-17:30，这代表了你的默认工作时间。
2. Rest 这是多个连续的时间段，默认为 12:00-13:30 / 17:30-18:00，在这个时间段内将不会记录工作时间
3. Core 这是一个连续的时间段，默认为 09:00-17:30，这代表了你在这段时间内必须在岗。

我们不妨翻译借助默认时间来解读一下这个规则：
你的工作是弹性制的，默认工作时间是Work，由于扣除了Rest，所以你的工作时间是每天8小时。当然，由于Core的存在，实际上你的弹性工作制的弹性所在仅仅是你可以在8:00-9:00之间的任意时间点开始上班。
此外，你由于弹性上班机制可以晚一点到岗，所以你需要在后面补全晚到岗的时间。比如你在8:37上班，那你就得多补37分钟。所有在**非Rest且非Work**时段进行的工作都可以用于弥补这段时间。

并且你并不需要在当日就弥补完这段时间，保险起见，你只需要在**该日所在的自然月**内弥补完这段时间即可，换句话说就是你的日均净上班时长达到了8h。
需要注意的是，你每个月的工资会被打散到每一分钟里面。也就是说你每分钟的工资=本月工资/（当月法定工作日数量*8(h)*60(min)）。
在每月结算时，你少工作的每一分钟，都**会**被扣除相应的工资。而你多工作的每一分钟，都**不会**被奖励额外的工资。

额外地，你可以请假，在本工具中，请你直接输入请假的分钟数。请自行忽略你的请假时段中Rest的部分。同时，将你的请假时间同样视为已打卡。

本工具实现了以下内容：
1. 根据你的打卡记录实时计算你目前的上班时长溢出或仍需弥补多少。
2. 根据你当前的仍需要弥补的分钟数量，和你请假的时长，根据你的本月工资自动计算你的预计本月工资。
3. 在你输入后，自动保存你所输入的信息，在每次启动后可以快速读取以往的状态，并进行修改。

你需要提供的信息如下:
1. 在全局设置中，设置每月工资，调整Work，Core的时间段，并添加Rest的时间段。
2. 进入日历界面后，在点击到对应日期上时，给出对应日的打卡记录。
3. 在日历页面的下方，输入你的请假时长（分钟）
